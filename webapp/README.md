# CAMT TA Timesheet

ระบบบันทึกเวลาทำงานผู้ช่วยสอน (TA Timesheet) สำหรับ CAMT — เว็บแอปพลิเคชัน Next.js + Supabase (PostgreSQL) พร้อม deploy บน Vercel

> **สำคัญเรื่องเวอร์ชัน / ความปลอดภัย:**
> - โปรเจกต์นี้ใช้ **Next.js 14.2.x (webpack)** — `package.json` จำกัดไว้ที่ `^14.2.33` (ต่ำกว่า 15) เพราะ Turbopack ของ Next 16 มีบั๊กพาร์ส ESM ของ `@supabase/postgrest-js` ("Unterminated block comment") เวอร์ชัน 14.2.33 ปิดช่องโหว่ความปลอดภัยสำคัญของ Next แล้ว (รวมถึง middleware auth bypass)
> - **อย่ารัน `npm audit fix --force`** เด็ดขาด เพราะมันจะดัน `next` ขึ้น 16 (พังเรื่อง Turbopack) และ downgrade `exceljs` เป็น 3.x (breaking)
> - วิธีอัปเดตให้ปลอดภัยคือ **clean reinstall**: ลบ `node_modules` + `package-lock.json` แล้ว `npm install` — จะได้ Supabase 2.74+ (ปิดช่องโหว่ auth-js), Next 14.2.33, postcss 8.5.10+ โดยอัตโนมัติ
> - ช่องโหว่ที่เหลือ (uuid ใน exceljs) เป็นระดับต่ำและไม่กระทบการใช้งานนี้ (เราไม่ได้ส่ง buffer ให้ uuid)

นำเข้าข้อมูลตั้งต้นจากไฟล์ CSV ภาคการศึกษา **2569/1** เรียบร้อยแล้ว (ผู้ช่วยสอน 48 คน, วิชา 39 วิชา, section 77 รายการ)

## ฟีเจอร์

**1. เข้าสู่ระบบ** — ใช้ **อีเมล + รหัสผ่าน** โดยรหัสผ่านเริ่มต้นคือ **เบอร์โทรศัพท์** (ถ้าไม่มีเบอร์ = `0123456789`). บัญชี user ทั่วไป **สร้างจากการนำเข้าไฟล์โดยผู้ดูแล** (ปิดการลงทะเบียนเอง)

**2. ระบบผู้ดูแล (Admin)**
- **นำเข้าข้อมูล (Import)** จากไฟล์ CSV รูปแบบเดียวกับที่ใช้งานจริง — ไฟล์ผู้ช่วยสอน → สร้าง/อัปเดต user, ไฟล์ EMS → สร้างวิชา/section/มอบหมายงาน (จับคู่ TA อัตโนมัติ) อัปโหลดซ้ำได้โดยไม่สร้างข้อมูลซ้ำ และไม่เปลี่ยนรหัสผ่าน user เดิม
- **ลบข้อมูล / รีเซ็ตระบบ** (แท็บ "นำเข้า/ลบ/สำรองข้อมูล") — ลบ user (ยกเว้น admin), วิชา, section, การมอบหมายงาน และ timesheet ทั้งหมด ต้องพิมพ์ยืนยัน "RESET" ก่อน (เก็บ admin/หลักสูตร/การตั้งค่าไว้)
- **ปีการศึกษา (รองรับหลายปี)** — แท็บ "ปีการศึกษา" สร้างปีใหม่ (เริ่มจากว่าง) → สลับปีที่ใช้งาน → แล้วค่อย import ข้อมูลของปีนั้น ข้อมูลแต่ละปีแยกกัน ปีเก่าถูกเก็บไว้และดู/ออกใบเบิกย้อนหลังได้ (เลือกปีในแท็บ Timesheet)
- **จัดการข้อมูล (CRUD)** — เพิ่ม/แก้ไข/ลบ ได้: ผู้ใช้ (แท็บ "ผู้ใช้" + รีเซ็ตรหัสผ่าน), วิชา/section และการมอบหมายผู้ช่วยสอน (แท็บ "วิชา/Section"), และแก้/ลบรายการ timesheet รายตัว (ในแท็บ Timesheet)
- **สำรองข้อมูล (Backup)** — ดาวน์โหลดสแนปช็อตทั้งระบบเป็น **.json** (ครบทุกตาราง สำหรับกู้คืน) หรือ **.xlsx** (อ่านง่าย แยก sheet ต่อขั้นตอน)
- **ตั้งค่าการคิดเงินทุน ป.ตรี** (แท็บ "นำเข้า/ลบ/สำรองข้อมูล") — ปรับ **อัตรา (บาท/ชม.)** และ **เพดานชั่วโมง/วัน** ที่ใช้แปลงยอดเงินตอนออกใบเบิกของนักศึกษาทุน ป.ตรี ได้เอง (ค่าเริ่มต้น 50 บาท/ชม. ไม่เกิน 8 ชม./วัน) การทบชั่วโมงจะข้ามวันหยุด (blackout) ให้อัตโนมัติ
- กำหนด **ช่วงวันที่ห้ามลงเวลาทำงาน** (blackout) และเลือกได้ว่าจะมีผลกับ **หลักสูตรใดบ้าง** (ไม่เลือก = ทุกหลักสูตร)
- ดู **timesheet ของทุกคน** แยกตามเดือน กรองตามหลักสูตร/ประเภทการจ้าง และ **ส่งออก CSV**
- **ออกใบเบิกแยกไฟล์ต่อ section (.zip)** — สำหรับคนที่มีหลายวิชา/section โดยเฉพาะ ทุน ป.ตรี ระบบจะกระจายชั่วโมง (50 บาท/ชม., ≤8 ชม./วัน) แบบ **ไม่ชนวันกันข้ามทุก section** (ใช้เพดานวันร่วมกัน) แล้วแยกเป็น 1 ไฟล์ต่อ section รวมเป็น .zip
- **ออกใบเบิกรายเดือนเป็น .xlsx** ของผู้ช่วยสอนแต่ละคน หัวเรื่องเปลี่ยนตามประเภท: TOR (จ้างเหมา) = "แบบใบเบิกค่าตอบแทนงานจ้างเหมาผู้ช่วยสอน", นักศึกษา (ทุน ป.ตรี / TA-RA) = "แบบใบเบิกค่าตอบแทนทุนนักศึกษาผู้ช่วยสอน" มีคอลัมน์ รหัสวิชา/Section/หลักสูตร/ชื่อ-นามสกุล/วันที่/จำนวนชั่วโมง/เงินต่อชั่วโมง/รวมเป็นเงิน/หมายเหตุ และ **ช่องลายเซ็นทุกแถว** พร้อมแถวรวม และ footer 4 ช่อง (ผู้จัดทำ/ผู้ตรวจสอบ · หัวหน้าภาควิชาหรือตำแหน่งอื่นที่เทียบเท่า · ผู้อนุมัติ · ผู้จ่ายเงิน) — ผู้ใช้ทั่วไปดาวน์โหลดใบเบิกของตนเองได้จากหน้า dashboard

**3. ระบบผู้ใช้ (กรอก timesheet)**
- เลือก **รายวิชา + section** ที่ตนเองรับผิดชอบ
- เลือก **วันที่** ทำงานจากปฏิทินรายเดือน พร้อม **หมายเหตุ (remark)** ที่ไม่บังคับ
- วันที่อยู่ในช่วง blackout หรือนอกภาคเรียนจะถูกล็อกไม่ให้เลือก
- ผู้ใช้แบ่งเป็น 3 ประเภท: **TOR (จ้างเหมา)**, **ทุน ป.ตรี**, **TA/RA**

**4. สรุปชั่วโมง / ค่าจ้าง + ควบคุมงบประมาณ**
- ผู้ใช้เลือกแค่ "วันที่" — ระบบคำนวณ **จำนวนชั่วโมง** จากเวลาเริ่ม–เวลาจบของ section และ **ค่าจ้าง** จาก ชั่วโมง × อัตราค่าจ้าง/ชม. (จริง)
- มี **สรุปรายเดือน** (จำนวนวัน / ชั่วโมง / ค่าจ้างรวม) ทั้งฝั่งผู้ใช้และผู้ดูแล
- การลงเวลาจะ **ไม่เกิน "ค่าใช้จ่ายคาดการณ์"** ของ section นั้น — แถบงบประมาณแสดงยอดที่ใช้/คงเหลือ และระบบบล็อกเมื่อจะเกินงบ (ตรวจทั้งฝั่ง client และ server)

## โครงสร้างโปรเจกต์

```
webapp/
├── supabase/
│   ├── schema.sql      # โครงสร้างฐานข้อมูล (รันก่อน)
│   └── seed.sql        # ข้อมูลตั้งต้นจาก CSV + รหัสผ่าน (รันทีหลัง)
├── src/
│   ├── app/
│   │   ├── login/ register/        # หน้าเข้าสู่ระบบ / ลงทะเบียน
│   │   ├── dashboard/              # หน้ากรอก timesheet ของผู้ใช้
│   │   ├── admin/                  # หน้าผู้ดูแล (timesheet + blackout)
│   │   └── api/                    # API routes (auth, timesheet, admin)
│   ├── lib/                        # supabase client, auth (JWT/bcrypt), constants
│   ├── components/                 # NavBar
│   └── middleware.js               # ป้องกันเส้นทาง /dashboard, /admin
├── SAMPLE_LOGINS.md                # บัญชีตัวอย่างสำหรับทดสอบ
└── .env.example
```

## การติดตั้ง (ทีละขั้น)

### 1) สร้างฐานข้อมูลบน Supabase
1. สร้างโปรเจกต์ที่ https://supabase.com
2. เปิด **SQL Editor** → วางเนื้อหา `supabase/schema.sql` แล้ว Run
3. เปิด **SQL Editor** อีกครั้ง → วางเนื้อหา `supabase/seed.sql` แล้ว Run

> หากเคยรัน schema เวอร์ชันก่อนหน้าไปแล้ว ให้รัน `schema.sql` ใหม่ (จะ DROP ตารางเดิมทั้งหมด) เพราะมีการเพิ่มตาราง `terms` และคอลัมน์ `semester` ใน `timesheet_entries`

### 2) ตั้งค่า Environment Variables
คัดลอก `.env.example` เป็น `.env.local` แล้วกรอกค่า:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<service_role secret จาก Project Settings → API>
JWT_SECRET=<ข้อความสุ่มยาว ๆ เช่น `openssl rand -base64 32`>
```

> ⚠️ `SUPABASE_SERVICE_ROLE_KEY` ใช้ฝั่งเซิร์ฟเวอร์เท่านั้น ห้ามเปิดเผยใน client

### 3) รันบนเครื่อง
```bash
npm install
npm run dev
```
เปิด http://localhost:3000

### 4) Deploy บน Vercel
1. push โค้ดขึ้น GitHub แล้ว Import เข้า Vercel
2. ใส่ Environment Variables 3 ตัวข้างบนใน Vercel → Settings → Environment Variables
3. Deploy

## หมายเหตุด้านความปลอดภัย / สถาปัตยกรรม
- รหัสผ่านถูกเข้ารหัสด้วย **bcrypt**; เซสชันเป็น **JWT** เก็บใน httpOnly cookie
- API routes ทั้งหมดเรียก Supabase ผ่าน **service role key** ฝั่งเซิร์ฟเวอร์ และตรวจสอบสิทธิ์ (user/admin) ก่อนทุกครั้ง
- ตาราง: `curricula, users, courses, sections, assignments, blackout_periods, blackout_curricula, timesheet_entries, settings`
- การลงเวลา 1 แถว = (ผู้ใช้ + section + วันที่) มี unique constraint กันบันทึกซ้ำ

## ที่มาของรหัสผ่านในข้อมูลตั้งต้น
สร้างจากไฟล์ `ผู้ช่วยสอน_2569-1.csv` โดย: **รหัสผ่าน = เบอร์โทรศัพท์** ของแต่ละคน ถ้าไม่มีเบอร์ = `0123456789`

หากต้องการเปลี่ยนรหัสผ่านของ user ที่มีอยู่แล้วให้เป็นเบอร์โทร ให้ไปที่ admin → แท็บ "นำเข้า/ลบ/สำรองข้อมูล" → ปุ่ม "รีเซ็ตรหัสผ่านผู้ใช้เป็นเบอร์โทร"
